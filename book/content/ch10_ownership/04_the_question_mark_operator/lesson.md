In the previous lesson, we explored Automatic Cleanup. Now we are going to take a look at The Question Mark Operator, which is an important concept in Krypton. As you may already be aware, many programming languages have some way of talking about using early returns in functions that also hold owned resources, and Krypton is no exception to that general pattern. It is worth mentioning, however, that the exact spelling still matters, because small syntax choices tend to shape how comfortably a reader can move through later code.

The example below demonstrates that the same `?` operator from error handling works here, but Krypton also keeps track of which owned resources must still be cleaned up on the way out. Note that this combination is a large part of the pitch for the ownership model, because it aims to keep error paths readable instead of forcing explicit cleanup code into every branch. This may sound somewhat ordinary, and in a sense it is, but that is also why it is useful to see it written out plainly before the lessons get more layered. Interestingly, this is somewhat similar to error propagation plus scope cleanup working together, although the Krypton version has its own punctuation and naming choices that should be read carefully.
