In the previous lesson, we explored Type Annotations. Now we are going to take a look at Koan Types, which is an important concept in Krypton. As you may already be aware, many programming languages have some way of talking about using the basic value forms from the chapter in a small repair exercise, and Krypton is no exception to that general pattern. It is worth mentioning, however, that the exact spelling still matters, because small syntax choices tend to shape how comfortably a reader can move through later code.

The example below demonstrates that the blanks force the reader to choose values whose types match the annotations already written in the file. Note that koans are intentionally a little incomplete so the reader has to supply the missing pieces rather than only reading along passively. This may sound somewhat ordinary, and in a sense it is, but that is also why it is useful to see it written out plainly before the lessons get more layered. Interestingly, this is somewhat similar to small fix-the-example exercises in tutorial material, although the Krypton version has its own punctuation and naming choices that should be read carefully.
