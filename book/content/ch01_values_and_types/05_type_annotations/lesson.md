In the previous lesson, we explored Unit. Now we are going to take a look at Type Annotations, which is an important concept in Krypton. As you may already be aware, many programming languages have some way of talking about writing types explicitly on bindings when you want to pin down what a value should be, and Krypton is no exception to that general pattern. It is worth mentioning, however, that the exact spelling still matters, because small syntax choices tend to shape how comfortably a reader can move through later code.

The example below demonstrates that a binding annotation appears after the name as in `let count: Int = 3`. Note that the example stays repetitive on purpose because annotations are mostly a matter of reading the punctuation until it feels visually normal. This may sound somewhat ordinary, and in a sense it is, but that is also why it is useful to see it written out plainly before the lessons get more layered. Interestingly, this is somewhat similar to variable annotations in typed languages that place the type next to the name, although the Krypton version has its own punctuation and naming choices that should be read carefully.
