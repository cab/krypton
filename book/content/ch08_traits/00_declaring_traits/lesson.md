In the previous lesson, we explored Koan Generics. Now we are going to take a look at Declaring Traits, which is an important concept in Krypton. As you may already be aware, many programming languages have some way of talking about introducing a shared interface that multiple types can implement, and Krypton is no exception to that general pattern. It is worth mentioning, however, that the exact spelling still matters, because small syntax choices tend to shape how comfortably a reader can move through later code.

The example below demonstrates that a trait declaration names the type parameter it abstracts over and lists the required methods inside braces. Note that traits are easier to understand when paired immediately with a tiny type and instance, because the abstract interface then has an obvious concrete target. This may sound somewhat ordinary, and in a sense it is, but that is also why it is useful to see it written out plainly before the lessons get more layered. Interestingly, this is somewhat similar to type classes or traits in languages with ad hoc polymorphism, although the Krypton version has its own punctuation and naming choices that should be read carefully.
