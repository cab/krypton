In the previous lesson, we explored Generic Functions. Now we are going to take a look at Generic Types, which is an important concept in Krypton. As you may already be aware, many programming languages have some way of talking about data types that hold values of many possible element types, and Krypton is no exception to that general pattern. It is worth mentioning, however, that the exact spelling still matters, because small syntax choices tend to shape how comfortably a reader can move through later code.

The example below demonstrates that a parameterized type declaration such as `type Box[a] = Box(a)` names a family of related types rather than one single concrete type. Note that this becomes much easier to internalize once the reader sees that the generic parameter sits on the type declaration for the same reason it sits on a generic function signature. This may sound somewhat ordinary, and in a sense it is, but that is also why it is useful to see it written out plainly before the lessons get more layered. Interestingly, this is somewhat similar to generic container definitions in typed languages, although the Krypton version has its own punctuation and naming choices that should be read carefully.
