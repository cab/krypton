In the previous lesson, we explored Blocks. Now we are going to take a look at If Expressions, which is an important concept in Krypton. As you may already be aware, many programming languages have some way of talking about using `if` and `else` as ordinary expressions that return values, and Krypton is no exception to that general pattern. It is worth mentioning, however, that the exact spelling still matters, because small syntax choices tend to shape how comfortably a reader can move through later code.

The example below demonstrates that Krypton writes conditional branches as `if cond { ... } else { ... }`, and each branch produces a value of the same overall type. Note that the important point is not the novelty of conditionals themselves but the fact that they fit directly inside larger expressions instead of sitting apart as a separate statement form. This may sound somewhat ordinary, and in a sense it is, but that is also why it is useful to see it written out plainly before the lessons get more layered. Interestingly, this is somewhat similar to expression-based conditionals in functional languages, although the Krypton version has its own punctuation and naming choices that should be read carefully.
